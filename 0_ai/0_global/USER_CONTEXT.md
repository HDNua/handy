## 2. Key Project Asset: DataMover (DM) Verification Framework
*   **Project Name**: DMA (DataMover) Protocol & Data Integrity Verification Automation.
*   **The Hardware Context (Domain Knowledge)**:
    *   **DataMover**: A dedicated DMA controller handling direct memory access between Modem IPs (Clients) and DRAM to offload CPU.
    *   **Protocol Complexity**: 
        *   **Flow Control**: Uses Ready/Valid handshake. Secure but can throttle the Client.
        *   **Non-Flow Control**: Streaming mode without backpressure. risky (overwrite potential) but fast. Requires strict interval guarantees.
    *   **Domain Constraints**: 
        *   Transmission Interval Max: **1ms (1 TTI)**.
        *   Non-Flow Control clients must guarantee a minimum interval (e.g., 20us).
    *   **Verification Strategy (The "4 Material" Approach)**:
        1.  **Client Data (`*_DM_test.txt`)**: Golden Reference generated by Checker IP.
        2.  **Simulation Log**: General Modem TOP simulation log containing DM Interrupt events.
        3.  **AXI Transaction Log (`dram_cp2ap.txt`)**: Cycle-accurate AXI signal dump.
            *   *Challenge*: Data arrives in fragmented bursts with byte-enable signals (`STRB`).
            *   *Solution (s03_get_dram_data)*: Implemented a **Byte-merging & Logic Reconstruction** algorithm in Python to re-assemble fragmented AXI bursts into coherent 64-byte blocks.
        4.  **DM Spec**: Configuration map (Base Addr, Flow Control Mode, Interrupt Mapping).
*   **The Problem**:
    *   Legacy **Perl** scripts were "Write-Only Code" (unmaintainable).
    *   Verification relied on "Eye-Checking" simulation waveforms, missing complex timing violations.
    *   **Specific Pain Point**: Flow Control Clients generating interrupts too frequently caused the legacy script to desynchronize (Error 2.2).
*   **The Solution (Architecture)**:
    *   **Python Migration**: Re-architected the flow using **OOP** (Parser-Runner) with modules `s00` to `s15`.
    *   **Automated Error Signature Analysis**:
        *   `INTERVAL_ERROR` (s11): Violation of minimum gap in Non-Flow clients (Potential FIFO Overflow).
        *   `DATA_UNMATCHED` (s12): The core data integrity check.
        *   **Heuristic Timing Checks (s15)**: Implemented adaptive interval checking (e.g., 1200ns vs 1500ns) to filter false positives in tight-loop scenarios where simulation time vs. real time drifts.
        *   `NO_DATA_IN_MEM`: Data Loss detection (Client sent, but DRAM didn't receive).
        *   `NO_INTERRUPT` / `NO_BURST`: Protocol Handshake failure (Missing Pairing).
    *   **Engineering Efficiency**:
        *   **DX**: Adopted **VSCode + TDD** environment, reducing debug time by **50%+**.
        *   **Optimization**: Implemented `fseek` tail-reading for GB-sized logs.
*   **The Achievement**:
    *   Transforming a "Black-box legacy script" into a **"Transparent, Maintainable Verification Framework."**
    *   Secured **Zero-Defect** regarding DMA data corruption issues in subsequent projects (Project 7, etc.).
