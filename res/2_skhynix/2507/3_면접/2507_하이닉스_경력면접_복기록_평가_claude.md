## 1. 전반적 구조 문제

### 문제
PT 10분 + Q&A 20분 형식임에도 불구하고, 30분 전체를 발표로 소진함.
면접관이 중간에 질문을 끼워 넣는 방식으로 진행됨.

### 원인 분석

"10분 PT"라는 규격을 실제로 연습하지 않은 것으로 보임.

### 교훈
- 발표 슬라이드는 **10분 내에 끝나도록 설계**해야 함
- 나머지 20분은 면접관이 원하는 방향으로 대화하는 시간
- 면접관이 "빨리 해달라"고 할 때는 이미 늦은 것

## 2. P6 Register Functional Coverage 답변 — 가장 큰 실패 포인트

### 상황
면접관이 가장 관심을 가진 주제.
"어려움이 많았을텐데 어떻게 문제를 해결했는지" 질문.

### 문제가 된 답변 흐름
1. 1차 답변: "담당자와의 합의로 해결" → 면접관: "혹시 의사소통 관련 답변인가요?"
2. 2차 답변: "sqlite3 도구 사용으로 성능 향상"

### 왜 실패했는가

**첫 답변이 엉뚱한 방향으로 출발**
- 면접관이 기대한 것: "이 기술 문제를 어떻게 해결했는가"
- 실제 답변: "커뮤니케이션으로 해결"
- 면접관이 직접 "기술적 방법은?" 이라고 재질문해야 했음 → 면접관의 에너지를 소진시키는 흐름

**sqlite3 답변이 공중에 뜸**
- Register Functional Coverage 개발에서 sqlite3가 왜 필요했는지, 어떤 문제를 해결했는지 설명이 이어지지 않음
- 면접관 입장에서는 "갑자기 sqlite3?" 라는 반응이 자연스러움

### 올바른 답변 구조 (STAR)

| 단계 | 내용 |
|------|------|
| **Situation** | 레지스터 조합의 경우의 수가 기하급수적으로 많아 전수 검사가 불가능 |
| **Task** | 현실적으로 검증 가능한 범위를 선별해야 함 |
| **Action** | (1) Spec 기반으로 의미있는 조합 분류 (2) sqlite3로 testcase DB화하여 중복 제거 및 취합 성능 향상 (3) 담당자와 scope 합의 |
| **Result** | RTL bug 1건 탐지, Coverage Closure 달성 |

> **핵심:** 기술적 내용을 먼저, 협업 내용은 그 다음.
> sqlite3는 "성능"이 아니라 "대규모 조합 관리"를 위한 도구임을 명확히 설명.

## 3. VP (Virtual Platform) 답변 — 준비 부족

### 문제
VP는 사내 프로젝트라 면접관이 모를 수 있음 → 예상 가능한 상황.
그럼에도 "아는 선에서만 대답"하는 데 그침.

### 교훈
외부인에게 설명한다고 가정하고 사전에 정리해야 함.

> "VP란 무엇인가 / 왜 필요한가 / 내가 거기서 어떤 역할을 했는가"
> 이 세 가지를 1분 이내로 설명하는 스크립트 준비 필수.

## 4. DT 지적 — 포지셔닝 실패

### 면접관 발언
> "검증으로 지원하셨지만 실제 업무는 DT에 가깝다"

### 분석
자동화 도구 개발(ALT), LLM 환경 구축, C++ VP 개발 등 tool/script 위주의 업무가 강조되다 보니, **검증 엔지니어보다 DT 엔지니어로 인식됨.**

SK하이닉스가 채우려는 포지션은 "검증 전문가"인데, 지원자는 "자동화 전문가"로 어필한 셈.

### 교훈
- 자동화/스크립팅은 검증을 위한 **수단**임을 일관되게 강조해야 함
- "DM IP Spec 기반 검증 시나리오 수립", "Coverage Closure 달성" 같은 검증 본연의 업무를 전면에 내세워야 함
- 자동화는 **"검증 효율을 높이기 위한 수단"** 으로만 포지셔닝

## 5. 갈등 해결 답변 — 리스크 있는 표현

### 답변 내용
"한두 번이 아니다 → 한 번 강하게 어필 → 안 되면 대부분 따른다"

### 리스크
- "강하게 어필"이라는 표현이 면접관에 따라 부정적으로 들릴 수 있음
- "대부분 따른다"는 표현이 주체성 없는 사람으로 보일 수 있음

### 권장 방향
- "강하게"보다 **"구체적 근거를 들어 설득을 시도한다"** 로 표현
- 결국 따를 때도 **"팀 방향에 맞추되, 개선 제안은 계속한다"** 식으로 주체성 유지

## 6. 종합 평가

### 잘한 점
- P4 (Regression 8배 단축) 기술 답변은 명확하고 자신있게 답변함
- 면접 분위기 자체는 나쁘지 않았던 것으로 보임
- 솔직한 복기록 작성 → 다음 면접 준비에 실질적으로 유용함

### 아쉬운 점
- PT 시간 배분 실패 → 핵심 슬라이드들이 후반부에 몰려 급하게 처리됨
- P6에서 면접관이 가장 관심있던 주제를 제대로 답변하지 못함
- 자동화 업무 강조 → DT 포지셔닝으로 인식됨
- 사전 SK하이닉스 인재상(SUPEX, VWBE 등) 준비 안 됨

### 자체 평가와 일치
> "자신감이 과해 준비가 부실했던 것이 탈락 원인"

타당한 자기 분석. 이번에는 동일한 실수를 반복하지 않는 것이 핵심.
