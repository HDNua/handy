DMA / Bus / DRAM 감각 잡기

버튼을 눌러서 CPU가 직접 옮기는 경우와 DMA가 버스를 타고 DRAM에 쓰는 경우를 비교해보세요. 아래 그림은 개념을 직관적으로 보여주기 위한 단순화입니다.

모드 선택

핵심:
• CPU 직접 복사 = IP → CPU → BUS → DRAM
• DMA 사용 = IP → DMA → BUS → DRAM
• 버스 = 주소 + 데이터 + 제어 + 핸드셰이크

지금 문맥에서는 버스를 거의 주소 기반 접근 통로로 보면 됩니다. 즉 누군가가 addr, data, write/read, valid/ready 같은 신호를 내보내며 공용 통로를 쓰고 있으면, "아 이게 버스구나"라고 느끼면 됩니다.

짧은 해설

CPU가 IP의 데이터를 읽고 다시 DRAM에 써주는 그림입니다. 되긴 되지만, 데이터가 크면 CPU가 단순 복사 노동을 많이 하게 됩니다.
1) CPU가 직접 복사 (CPU = bus master)
IP → CPU → BUS → DRAM 경로입니다. CPU도 버스를 통해 DRAM에 씁니다. DMA와 차이는 "버스를 쓰냐 마냐"가 아니라 "누가 버스를 drive하냐"입니다.
