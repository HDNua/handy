# UART 개념 정리 메모

작성일: 2026-03-29

---

## 1. 한 줄 정의

UART는 주로 서로 다른 장치 간 데이터 통신을 위해 사용되는 비동기 직렬 통신 프로토콜이며,
그 프로토콜을 구현하는 하드웨어를 가리키기도 한다.

조금 더 풀어 쓰면:

- `UART 통신`은 규칙이다.
- `UART TX / UART RX`는 그 규칙대로 실제 송수신을 수행하는 하드웨어 블록이다.

즉, `UART`라는 말은 다음 두 의미로 같이 쓰인다.

- 통신 방식
- 그 통신 방식을 수행하는 송수신기

---

## 2. 왜 UART를 쓰는가

UART의 핵심 장점은 단순함이다.

- 선이 적게 든다
- 구현이 쉽다
- 디버깅이 쉽다
- 서로 다른 장치를 연결하기 좋다

병렬 버스처럼 8비트를 한 번에 보내려면 선도 많이 필요하고 타이밍도 까다롭다.
반면 UART는 느리더라도 `1비트 선`으로 순서대로 보내기 때문에 구조가 단순하다.

그래서 UART는 “고성능 통신”보다는 “간단하고 확실한 연결”에 더 강하다.

---

## 3. 언제 UART를 떠올리면 되는가

가장 자연스러운 맥락은 다음과 같다.

- 한 칩 내부의 고속 버스가 아니라
- 서로 떨어진 통신 상대끼리
- 적은 선으로
- 간단하게 데이터를 주고받고 싶을 때

대표 예시는 아래와 같다.

- MCU 보드 <-> PC
- 보드 <-> 센서 모듈
- SoC <-> 외부 디버그 콘솔
- 장비 bring-up용 로그 출력
- USB-UART를 통한 시리얼 터미널 연결

즉, UART는 “서로 다른 장치가 규칙을 맞춰 직렬로 통신하는 방식”으로 이해하면 된다.

---

## 4. 반드시 서로 다른 칩에서만 쓰는가

엄밀히 말하면 `반드시`는 아니다.

하나의 SoC 안에도 UART IP가 들어 있을 수 있다.
하지만 그 UART 블록은 보통 칩 내부 버스용이라기보다,
결국 칩 바깥 핀으로 나가서 외부 장치와 통신하는 용도로 쓰인다.

그래서 감각적으로는 이렇게 이해하면 편하다.

- 칩 내부 통합 버스: AXI / AHB / APB
- 칩 바깥 외부 장치 연결: UART

즉,

`UART는 주로 칩 내부보다는 칩 밖의 다른 통신 상대와 연결할 때 더 자연스럽다.`

---

## 5. UART 프레임을 어떻게 이해해야 하는가

CPU가 UART TX에 `1바이트(8비트)`를 넘기면,
UART TX는 이 데이터를 그대로 한 번에 보내지 않고 프레임으로 감싸서 직렬화한다.

한 프레임의 기본 구조:

```text
start bit + data 8bit + stop bit
```

예:

```text
idle(1) -> START(0) -> D0 -> D1 -> D2 -> D3 -> D4 -> D5 -> D6 -> D7 -> STOP(1)
```

중요한 점:

- `start bit`는 “지금부터 1바이트 전송 시작”
- `stop bit`는 “이 1바이트 프레임 전송 끝”

즉, stop bit는 전체 메시지 끝이 아니라 `한 바이트 끝` 표시다.

문자열 `"Hello"`는 한 번에 보내는 것이 아니라,
`'H'`, `'e'`, `'l'`, `'l'`, `'o'`를 각각 한 프레임씩 보낸다.

## 6. 송신기와 수신기는 무엇을 하는가

### 6.1 UART TX

UART TX는:

- CPU에게서 1바이트를 받는다
- 그것을 `start/data/stop` 형식으로 직렬화한다
- `TX_SERIAL` 한 줄로 1비트씩 내보낸다

### 6.2 UART RX

UART RX는:

- 선이 idle 상태(1)인지 보고 있다가
- start bit의 하강을 감지한다
- 미리 약속된 baud rate에 맞춰 비트를 샘플링한다
- data 8bit를 다시 1바이트로 조립한다

즉, 실제 수신기 역할을 하는 것은 `UART RX`다.

---

## 7. Testbench의 monitor는 무엇인가

여기서 `monitor`는 디스플레이 모니터가 아니다.

검증 환경(UVM/TB)에서 monitor는:

- DUT 출력 신호를 관찰하고
- 그 신호를 해석해서
- transaction 형태의 데이터로 복원하는 블록이다

이번 UART TX 실습에서는 `tb_top.sv`의 `uart_monitor`가
실제 시스템의 UART RX가 하는 일을 흉내 낸다.

즉:

- 실제 시스템: `UART RX`
- Testbench: `monitor`

로 역할을 구분해야 한다.

scoreboard는 monitor가 복원한 결과가
원래 기대값과 같은지 비교해서 PASS/FAIL을 판단한다.

## 8. Receiver PC, Receiver Device, UART RX를 어떻게 구분해야 하는가

초기 설명에서는 `Receiver PC`라는 표현을 썼지만,
이 표현은 여러 오해를 만들 수 있다.

- 왼쪽 CPU도 하나의 PC처럼 읽힐 수 있음
- 화면 모니터와 혼동될 수 있음
- 장치 전체와 UART RX 블록이 분리되어 보이지 않음

그래서 더 좋은 표현은 다음과 같다.

- `Receiver Device`: 수신하는 상대 장치 전체
- `UART RX`: 그 장치 안에서 실제 수신을 담당하는 블록

그림은 아래처럼 이해하는 것이 가장 명확하다.

```text
[ CPU ]
    v
[ UART TX ] -> serial line -> [ UART RX ] -> [ Receiver Device 내부 처리 ]
```

혹은 더 간단히:

```text
[ CPU + UART TX ] -> [ UART RX inside Receiver Device ]
```

## 9. 칩, 보드, 장치는 어떻게 다른가

### 9.1 칩 (chip)

- 반도체 부품 하나
- 예: MCU, SoC, DRAM, 센서 IC, USB-UART 브리지 칩

### 9.2 보드 (board)

- 여러 칩과 부품을 올려서 연결한 PCB
- 예: 개발보드, FPGA 보드, 아두이노 보드, 메인보드

### 9.3 장치 (device)

- 실제로 기능을 수행하는 통신 상대 전체
- 상황에 따라 칩 하나일 수도 있고, 보드 하나일 수도 있고, PC 전체일 수도 있음

정리하면:

- 칩: 부품 1개
- 보드: 여러 부품을 연결한 판
- 장치: 실제 동작 주체 전체

---

## 10. 메인보드와 마더보드는 다른가

거의 같은 뜻이다.

- `motherboard`: 더 대중적으로 많이 쓰는 영어 표현
- `mainboard`: 제조사 문서나 수리 문맥에서 보일 수 있는 표현

한국어로는 둘 다 대체로 `메인보드`라고 부른다.

즉:

- CPU는 칩
- 메인보드는 보드
- PC 전체는 장치 또는 시스템

## 11. 이번 UART TX 실습에 적용하면

이번 `lv1_1_uart_tx` 실습을 개념적으로 보면 다음과 같다.

### 실제 시스템 관점

```text
CPU -> UART TX -> TX_SERIAL -> Receiver Device 안의 UART RX
```

### Testbench 관점

```text
sequence -> driver -> DUT(UART TX) -> monitor -> scoreboard
```

여기서:

- DUT는 UART TX
- monitor는 실제 UART RX 역할을 흉내 냄
- scoreboard는 복원 결과를 검증함

---

## 12. 최종 요약

이번 대화에서 정리한 핵심은 아래와 같다.

1. UART는 비동기 직렬 통신 규약이자, 그 규약을 구현하는 하드웨어 이름이기도 하다.
2. UART는 주로 서로 다른 장치 간 통신에 사용된다.
3. CPU가 UART TX에 준 1바이트는 start/data/stop 구조로 직렬 전송된다.
4. 실제 수신은 Receiver Device 내부의 UART RX가 수행한다.
5. Testbench의 monitor는 그 RX 동작을 흉내 내는 검증 블록이다.
6. `chip`, `board`, `device`는 서로 다른 범위의 개념이다.

이 수준까지 머릿속에 정리되어 있으면,
`UART TX RTL`, `UART RX 동작`, `monitor/scoreboard 역할`을 훨씬 덜 헷갈리고 볼 수 있다.
